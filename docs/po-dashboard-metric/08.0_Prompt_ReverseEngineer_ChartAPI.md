## Bối cảnh

App cần hiển thị 4 report "Qonversion report" theo **product**, lấy số **y như giao diện Analytics → Charts** của Qonversion (`dash.qonversion.io`):

1. **Subscriptions by Product** → Active Subscriptions + New Subscriptions
2. **New-User-to-Trial by Product** → New-User-to-Trial Conversion rate
3. **Trial-to-Paid by Product** → Trial-to-Paid Conversion rate + Revenue
4. **Refunds by Product** → Refunds + Refunds rate

Đã chốt: dùng **Chart API của dashboard Qonversion** (KHÔNG tự tính từ event). Crawler hiện tại ([`QonversionWebCrawlerClient`](../../backend/MediationPro.Infrastructure/Qonversion/QonversionWebCrawlerClient.cs)) chỉ gọi API **raw-data export** (`api/v1/exports`) — KHÁC với API đứng sau các **Chart** này. Việc của bạn là tìm ra API Chart thật.

Đọc trước: [`08_Qonversion_Product_Reports.md`](08_Qonversion_Product_Reports.md) toàn bộ (đặc biệt §1, §2, §3.2).

Bạn KHÔNG tự đăng nhập được Qonversion. Cần một trong hai:

- **(B)** Bạn có quyền truy cập trình duyệt (browser tool) + cookie dashboard hợp lệ (`QonversionAccount.DashboardCookie` + `DashboardAccountUid` trong DB) để tự mở chart và đọc Network.

Nếu cả A và B đều không có → STOP, báo cần HAR hoặc cookie+browser access. Không bịa endpoint.

## Việc cần làm

### 1. Với MỖI trong 4 chart, ghi lại từ Network:

| Mục cần ghi | Chi tiết |
|---|---|
| Chart name + vị trí nav | vd "New users → New-User-to-Trial Conversion" |
| URL đầy đủ + HTTP method | vd `GET https://dash.qonversion.io/api/v1/charts/...` |
| Request params / body | chart id/slug, `from_timestamp`/`to_timestamp` (đơn vị?), `platform`, `group_by` (giá trị để group theo **product** là gì?), `granularity`/`interval` (daily), `project`, `account`, `date_limiter_type` |
| Headers bắt buộc | Cookie, origin, referer, user-agent (so với `AddDashboardHeaders` hiện có) |
| **Response JSON shape** | dán mẫu rút gọn: cấu trúc theo ngày × product trông thế nào |
| **Có numerator/denominator?** | với chart tỷ lệ (New-User-to-Trial, Trial-to-Paid, Refunds rate): response có trả **count tử/mẫu** hay chỉ `%`? → quyết định cách gộp range |
| Product id field | tên field chứa product id trong response + ví dụ giá trị |
| Timezone | số liệu theo timezone nào (UTC hay timezone account)? |

### 2. Kiểm tra nút **"Chart data"** (export bảng dưới mỗi chart)
- Endpoint riêng? Trả CSV/JSON? Có đủ count (num/den) không? Thường bản export đầy đủ hơn chart → nếu có num/den thì ưu tiên dùng cho pipeline.

### 3. Kiểm tra có **API chính thức** không (ổn định hơn cookie-crawl)
- Đọc [`QonversionApiClient`](../../backend/MediationPro.Infrastructure/Qonversion/QonversionApiClient.cs) + `QonversionAccount.SecretKey`/`BaseUrl` (`https://api.qonversion.io/v3`).
- Tra tài liệu Qonversion (WebSearch/WebFetch) xem có **Data/Metrics/Analytics API** REST kèm API key cho các metric này không. Nếu có → khuyến nghị dùng thay cookie.

### 4. Đối chiếu định nghĩa metric (mục §2 + §10 của doc 08)
- "Active Subscriptions" khi group by product + Daily Qonversion định nghĩa chính xác ra sao? (đối chiếu chốt PO: "chỉ đếm trong range").
- "Refunds rate" mẫu số của Qonversion là gì? (đối chiếu chốt PO: refunds / purchases cùng product). Nếu khác → ghi rõ cả hai, ưu tiên số Qonversion.
