## 1. Sự thật đã verify (ground truth — KHÔNG đoán lại)

- **Endpoint thật:** `GET https://dash.qonversion.io/api/v1/analytics/chart/{chart_code}` — **cookie-authed** (KHÔNG phải `api.qonversion.io/v4`).
- **Response bọc** `{"success": true, "data": { ...chart... }}` → parse `.data`.
- **Auth:** tái dùng cookie dashboard có sẵn — `QonversionAccount.DashboardCookie` + `DashboardAccountUid`, pattern `AddDashboardHeaders` trong [`QonversionWebCrawlerClient`](../../backend/MediationPro.Infrastructure/Qonversion/QonversionWebCrawlerClient.cs). HttpClient của client này đã trỏ base `dash.qonversion.io` (client hiện gọi relative `api/v1/exports`).
- **Params:** `unit=day`, `segmentation=product_id`, `environment=1`, `project={projectKey}`, `currency=USD`, `from`/`to` (Unix giây).
- **7 chart code** (đã verify hỗ trợ `product_id`):

| metric_key | chart_code | measure | `total`/value nghĩa là |
|---|---|---|---|
| `active_subscriptions` | `active-subscriptions` | count | end-of-period |
| `new_subscriptions` | `new-subscriptions` | count | tổng range |
| `new_user_to_trial_rate` | `user-to-trial-conversion` | percent | wavg rate (+`totalWeight`) |
| `trial_to_paid_rate` | `trial-to-paid` | percent | wavg rate (+`totalWeight`) |
| `trial_to_paid_revenue_usd` | `proceeds` | usd | tổng range |
| `refunds_usd` | `refunds` | usd | tổng range |
| `refund_rate` | `refund-rate` | percent | wavg rate (+`totalWeight`) |

> ⚠️ Tên code phải chính xác: New-User-to-Trial = `user-to-trial-conversion` (có `-conversion`); Trial-to-Paid = `trial-to-paid` (KHÔNG `-conversion`, sai → HTTP 400).

- **series[]:** `label` (= product, chuỗi ghép `prod:store_id:...`), `total`, `totalPrev`, `totalWeight` (nullable, chỉ chart percent), `totalPrevWeight`, `data[]{start_time(Unix giây), value}`.
- **percent là số % trực tiếp** (`0.31`=0.31%, `20`=20%). KHÔNG nhân/chia khi LƯU; chỉ khi tính numerator ở 8.2/8.3.

## 2. Chiến lược pull — PER-DAY (đã chốt)

Gọi **mỗi ngày một request** (`from`/`to` = đúng 1 ngày UTC, `unit=day`) cho mỗi (project, platform, chart_code). Lý do: `totalWeight` là theo range query → per-day mới có denominator đúng từng ngày để 8.3 gộp range `Σnum/Σden`. Mỗi response per-day → `series[].data[]` 1 điểm.

Mirror cửa sổ trượt của crawler hiện có: dùng rolling window (vd T-3..T-1 hoặc cấu hình `lookbackDays`), lặp từng ngày.

## 4. Bronze schema — `bronze.qonversion_chart_metrics_raw`

Long format, 1 dòng / (ngày × project × platform × chart × product):
```
ingested_at        DATETIME,
source_channel     VARCHAR(32),     -- 'chart_crawler'
qon_project_key    VARCHAR(64),
platform           VARCHAR(16),
report_date        DATE,
chart_code         VARCHAR(64),
metric_key         VARCHAR(64),     -- map từ chart_code (bảng §1)
product_label      VARCHAR(255),    -- series[].label (raw)
metric_value       DECIMAL(20,6),   -- data point value của ngày (per-day pull → 1 điểm)
total_weight       DECIMAL(20,6) NULL,  -- series[].totalWeight (chart percent)
total_type         VARCHAR(10),     -- 'sum' | 'wavg'
measure            VARCHAR(10),     -- 'usd' | 'count' | 'percent'
currency           VARCHAR(8),
raw_payload        JSON,            -- response .data (hoặc series item) đã sanitize, KHÔNG chứa cookie
sync_batch_id      VARCHAR(64)
```
- DUPLICATE/PRIMARY KEY + partition theo tháng của `report_date` theo pattern bronze hiện có; dedup để 8.2 xử lý (giống `qonversion_events_raw`).
- Map `chart_code → metric_key` theo bảng §1.

## 5. Client — `QonversionChartApiClient`

- Gọi `GET api/v1/analytics/chart/{chartCode}?unit=day&segmentation=product_id&environment=1&project={project}&currency=USD&from={from}&to={to}`.
- Header: `AddDashboardHeaders` (Cookie, origin, referer `https://dash.qonversion.io/analytics/...`, user-agent). 401/403 → throw "cookie hết hạn" như client hiện có.
- Parse `{success,data}`; nếu `success=false` hoặc HTTP 400 (chart code/segmentation không hợp lệ) → log cảnh báo + bỏ qua chart đó (không làm hỏng cả batch).
- Trả về list rows (mỗi series × data point) cho job ghi bronze.

## 6. Job — `QonversionChartCrawlerJob`

Mirror [`QonversionWebCrawlerJob`](../../backend/MediationPro.Jobs/QonversionWebCrawlerJob.cs):
- Lấy default `QonversionAccount` (cookie + accountUid); apps `qonversion_crawler=true` + `qonversion_params.projectKey` (dùng lại `TryGetQonProjectKeyFromParams`, `ResolvePlatforms`, `BuildWorkItemsFromApps`).
- Rolling window theo ngày UTC (config `lookbackDays`). Với mỗi (project, platform) × 7 chart_code × mỗi ngày: gọi client, gom rows.
- Lưu raw JSON vào MinIO prefix `raw/qonversion/charts/{date}/{project}_{platform}_{chart}.json` (sanitize).
- Ghi bronze qua `InsertChartMetricRowsAsync`.
- Tôn trọng rate limit: `SemaphoreSlim` giới hạn concurrency (config), có thể delay nhẹ giữa call.
- KHÔNG enqueue transform (8.2 chưa có) — chỉ tới bronze.
