> **Loại task:** IMPLEMENT CODE (frontend). Slice cuối của tính năng Qonversion Product Reports.
> **Tiền đề:** 8.3 đã có endpoint `GET /api/apps/{appId}/dashboard/qonversion-products?report=&range=`.
> Đọc trước: [`08_Qonversion_Product_Reports.md`](08_Qonversion_Product_Reports.md) §1, §6; [`08.3_Prompt_Provider_Endpoint.md`](08.3_Prompt_Provider_Endpoint.md) §2-3 (shape + cột).

## 1. Mục tiêu

Thêm section **"Qonversion report"** ở cuối tab Dashboard (sau Adjust report), gồm **4 bảng** theo product:

| Bảng | report | Cột |
|---|---|---|
| Subscriptions by Product | `subscriptions` | Product · Active Subscriptions · New Subscriptions |
| New-User-to-Trial by Product | `new_user_to_trial` | Product · Conversion rate (%) |
| Trial-to-Paid by Product | `trial_to_paid` | Product · Conversion rate (%) · Revenue ($) |
| Refunds by Product | `refunds` | Product · Refunds ($) · Refund rate (%) |

Mỗi bảng gọi endpoint với `report` tương ứng + `range` hiện tại của dashboard.

## 2. Response (từ 8.3)
```ts
{
  report: 'subscriptions' | 'new_user_to_trial' | 'trial_to_paid' | 'refunds',
  dateRange: DashboardDateRange,            // type sẵn có
  rows: Array<{
    productId: string,
    activeSubscriptions?: number | null,
    newSubscriptions?: number | null,
    conversionRate?: number | null,         // số % trực tiếp, vd 0.28 = 0.28%
    revenueUsd?: number | null,
    refundsUsd?: number | null,
    refundRate?: number | null,             // số %, vd 4 = 4%
  }>,
  meta: { warnings: string[] }
}
```
> `conversionRate`/`refundRate` đã là **số phần trăm** → hiển thị `value.toFixed(2) + '%'`, **KHÔNG ×100**.
> `productId` là chuỗi ghép raw (`prod:store_id:...`) — hiển thị raw (PO yêu cầu Product id). Có thể `title`/truncate cho gọn nhưng giữ giá trị đầy đủ.

## 4. Format (dùng/extend `format.ts`)
- Conversion rate / refund rate: `formatPercent(v)` → `45.60%` (1-2 chữ số thập phân; v đã là %).
- Revenue / refunds: `formatUsd(v)` → `$1,234.56`.
- Active/New subscriptions: `formatCount(v)` → số (≥10K → `12.3K`).
- `null`/thiếu → `—`.

## 5. Trạng thái rỗng / lỗi
- Nếu `meta.warnings` chứa `qonversion_charts_not_configured` **hoặc** `rows` rỗng → hiển thị empty state riêng cho từng bảng (vd "No Qonversion data for this app/range."), không vỡ layout.
- Loading skeleton + error state như các bảng khác.
- 4 bảng độc lập (mỗi bảng 1 query) → 1 bảng lỗi không kéo sập 3 bảng kia.
