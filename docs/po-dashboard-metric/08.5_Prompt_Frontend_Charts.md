## 1. Muc tieu

Them chart trong section **"Qonversion report"** voi truc X la ngay (`reportDate`). Moi chart cho phep nhin dong gop theo product qua tung ngay trong range hien tai.

| Report | Chart chinh | Metric chinh | Ghi chu |
|---|---|---|---|
| Subscriptions by Product | Stacked column by product | `newSubscriptions` | `activeSubscriptions` la stock/snapshot, khong stack voi `newSubscriptions`; hien line/summary rieng neu can |
| New-User-to-Trial by Product | Grouped column hoac line by product | `conversionRate` | Percent khong duoc stack; co the dung line/grouped bar cho top products |
| Trial-to-Paid by Product | Stacked column by product + optional rate line | `revenueUsd`; optional `conversionRate` | Revenue stack duoc, rate khong stack |
| Refunds by Product | Stacked column by product + optional rate line | `refundsUsd`; optional `refundRate` | Refund dollars stack duoc, rate khong stack |

Nguyen tac nghiep vu:

- **Stack chi ap dung cho additive metrics**: count flow / USD (`newSubscriptions`, `revenueUsd`, `refundsUsd`).
- **Khong stack percent metrics** (`conversionRate`, `refundRate`) vi tong % khong co y nghia.
- Product series gioi han top N de chart doc duoc; mac dinh **Top 5 products + Other** cho stacked chart.
- Tooltip phai show full raw `productId`, ngay, va gia tri format dung.

## 2. Backend endpoint moi

Them endpoint moi, khong thay doi endpoint aggregate 8.3:

```http
GET /api/apps/{appId}/dashboard/qonversion-products-series?report={report}&range={range}
report in subscriptions | new_user_to_trial | trial_to_paid | refunds
range  in today | yesterday | last7
```

### 2.1 Response shape

Dung row long-ish theo `reportDate + productId` de frontend pivot thanh Recharts data.

```ts
{
  report: 'subscriptions' | 'new_user_to_trial' | 'trial_to_paid' | 'refunds',
  dateRange: DashboardDateRange,
  rows: Array<{
    reportDate: string,              // yyyy-MM-dd account/display timezone theo range resolver
    productId: string,               // raw product id
    activeSubscriptions?: number | null,
    newSubscriptions?: number | null,
    conversionRate?: number | null,  // da la so %, vd 0.28 = 0.28%; KHONG x100 o FE
    revenueUsd?: number | null,
    refundsUsd?: number | null,
    refundRate?: number | null       // da la so %, vd 4 = 4%
  }>,
  meta: { warnings: string[] }
}
```

### 2.2 Source table

Doc tu silver per-day:

```txt
silver.qonversion_product_metrics_daily
```

Cot can dung:

```txt
report_date, admob_app_id, platform, product_id,
metric_key, metric_value, numerator, denominator
```

`metric_key` lien quan:

```txt
active_subscriptions
new_subscriptions
new_user_to_trial_rate
trial_to_paid_rate
trial_to_paid_revenue_usd
refunds_usd
refund_rate
```

### 2.3 Daily aggregation rules

Group backend theo `report_date + product_id` va gop moi platform cua app.

| report | metric_key | field response | Daily aggregation |
|---|---|---|---|
| `subscriptions` | `active_subscriptions` | `activeSubscriptions` | End-of-day/snapshot trong ngay; neu co nhieu platform thi SUM snapshot theo platform |
| `subscriptions` | `new_subscriptions` | `newSubscriptions` | SUM(metric_value) theo ngay/product |
| `new_user_to_trial` | `new_user_to_trial_rate` | `conversionRate` | SUM(numerator) / NULLIF(SUM(denominator),0) * 100 |
| `trial_to_paid` | `trial_to_paid_rate` | `conversionRate` | SUM(numerator) / NULLIF(SUM(denominator),0) * 100 |
| `trial_to_paid` | `trial_to_paid_revenue_usd` | `revenueUsd` | SUM(metric_value) theo ngay/product |
| `refunds` | `refunds_usd` | `refundsUsd` | SUM(metric_value) theo ngay/product |
| `refunds` | `refund_rate` | `refundRate` | SUM(numerator) / NULLIF(SUM(denominator),0) * 100 |

Bat buoc:

- Rate tra ve la **percentage points**; vi du `0.28` nghia la `0.28%`.
- Khong average cac daily rate bang trung binh cong trong backend.
- Missing metric trong ngay/product -> field null.
- Neu khong co data -> `rows: []` + warning `qonversion_charts_not_configured`, khong throw.
- Sort row theo `reportDate ASC`, sau do product stable theo primary metric desc neu can.

## 4. Frontend chart UX

Them chart vao section **"Qonversion report"**, giu 4 bang 8.4 ben duoi hoac trong cung report panel.

Layout de xuat:

```txt
Qonversion report
  [Daily chart: Subscriptions]      [Table: Subscriptions]
  [Daily chart: New-user-to-trial]  [Table: New-user-to-trial]
  [Daily chart: Trial-to-paid]      [Table: Trial-to-paid]
  [Daily chart: Refunds]            [Table: Refunds]
```

Hoac tren desktop 2 columns:

```txt
[chart + table: subscriptions]     [chart + table: new_user_to_trial]
[chart + table: trial_to_paid]     [chart + table: refunds]
```

Bang aggregate trong moi report panel chi hien toi da khoang **10 rows** trong viewport cua card. Neu response co hon 10 product rows, giu day du rows nhung dat table body trong vung `overflow-y-auto` de user scroll noi bo; khong de table keo dai lam vo layout dashboard. Header table nen sticky khi scroll.

### 4.1 Product selection / Top N

Chart daily phai pivot product thanh series. De chart khong roi:

- Mac dinh lay **Top 5 product** theo tong primary metric trong range.
- Cac product con lai gop vao series `Other` cho additive metrics.
- Voi percent metrics, **khong tao Other bang average tuy tien**. Chi hien Top 5 product; khong hien Other.
- Tooltip `Other` co the ghi "Other products"; product raw id chi show cho series product that.

Primary metric theo report:

| report | Primary metric de rank |
|---|---|
| `subscriptions` | SUM(`newSubscriptions`) fallback max/latest `activeSubscriptions` |
| `new_user_to_trial` | Weighted denominator neu backend expose; neu khong, sort theo AVG available `conversionRate` va ghi ro la display ranking |
| `trial_to_paid` | SUM(`revenueUsd`) fallback AVG `conversionRate` |
| `refunds` | SUM(`refundsUsd`) fallback AVG `refundRate` |

Neu ranking percent can chinh xac hon, backend co the bo sung optional hidden fields `numerator`/`denominator`; nhung UI khong bat buoc hien.

### 4.2 Chart types

Dung `recharts`, khong them dependency.

| report | Chart |
|---|---|
| `subscriptions` | Stacked `BarChart` X=`reportDate`, stackId=`newSubscriptionsByProduct`; optional line/secondary note cho active total neu co thoi gian |
| `new_user_to_trial` | `LineChart` hoac grouped `BarChart` X=`reportDate`, series=Top 5 product, Y=`conversionRate` |
| `trial_to_paid` | Stacked `BarChart` X=`reportDate`, series=Top 5 product + Other, Y=`revenueUsd`; optional small line chart ben duoi cho `conversionRate` Top 5 |
| `refunds` | Stacked `BarChart` X=`reportDate`, series=Top 5 product + Other, Y=`refundsUsd`; optional small line chart ben duoi cho `refundRate` Top 5 |

Neu lam MVP nhanh:

- Bat buoc: stacked column cho `newSubscriptions`, `revenueUsd`, `refundsUsd`.
- Bat buoc: non-stacked line/grouped chart cho `conversionRate` va `refundRate`.
- Optional: overlay active subscriptions / rate tren cung card.

### 4.3 Format

- Count: `formatCount`.
- USD: `formatUsd`.
- Percent: `formatPercent(value, 2)`; **KHONG x100**.
- Null/missing -> khong ve point/bar hoac hien `-` trong tooltip.
- X-axis date: compact label `MM-dd` hoac `dd/MM`; tooltip show full `yyyy-MM-dd`.

### 4.4 Empty / Loading / Error

- Warning `qonversion_charts_not_configured` -> empty state rieng: `No Qonversion daily data for this app/range.`
- `rows` rong -> empty state rieng.
- Loading lan dau -> skeleton.
- Error -> retry card rieng tung report.
- 4 report query doc lap; mot report loi khong lam sap report khac.
